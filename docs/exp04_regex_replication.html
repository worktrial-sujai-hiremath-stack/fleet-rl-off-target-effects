Exp 04 Replication — copy-N regex, 4 new seeds

Does the dose-response capability pattern (N=0/3/10) hold across seeds?

TL;DR — Replicated (directionally) All 12 arm-seed runs (3 arms × 4 seeds) finished cleanly. In-harness training reward shows copy_n0 (0.489 ± 0.034) > copy_n3 (0.358 ± 0.022) ≈ copy_n10 (0.346 ± 0.048) — a clean dose-response where padding HURTS training. Seed variance is 2–5pp within-arm, smaller than the ~13pp between-arm gap. Direction is consistent with the original presentation, but narrow/hard probes on the 12 new models haven't been run so the original's on-task 25→45% and off-task 44→51% numbers are not yet directly compared.

What was being replicated

The original Exp 04 (bonus): GRPO'd Qwen3.5-1.7B on a regex-fix env where each prompt was augmented with "IMPORTANT: Before proposing your fix, first write the broken line EXACTLY N times" for N ∈ {0, 3, 10}. The original single-seed result reported:

  • Narrow (on-task) accuracy: Base 25% → Q 30% → Q3 35% → Q10 45% (monotone increase in target-distribution pass rate).
  • Generalization (off-task regex) accuracy: Base 44% → Q 53% → Q3 43% → Q10 51% (non-monotone — Q3 drops BELOW base).
  • Interpretation: dose-dependent capability degradation — training on the padded-prompt distribution pushes out of the off-task regex skill at N=3.

Why replicate

  • The capability-regression claim (Q3 < Base) hinges on a 1pp difference — well within probe variance. Needs seed bands.
  • Non-monotone shape is surprising — worth checking if different seeds preserve the V-shape or flatten it.
  • Narrow 25→45 is a big effect. If it survives 4 seeds with std ≪ 10pp it's a solid claim.

What happened

4 sky jobs (seeds 7, 13, 29, 47), each running all 3 arms sequentially on one H200:8 cluster via run-exp7-all-arms.sh. All 4 jobs SUCCEEDED. Each cluster took ~2.5–3 hours with 1–2 spot-preemption recoveries. The per-arm script (fleet-1p7b-exp7-copy-n.sh) was patched with a ${SUFFIX:-} plumb so seed replicates wrote to scoped S3 paths (exp7-copy_n{0,3,10}_seed{7,13,29,47}/) and didn't collide with the original SEED=42 run.

Observation: pad-N lowers cold-start AND final training reward. The pad-3 and pad-10 arms are essentially equal in in-harness training performance (~0.35) and both lose ~13pp to the unpadded arm. Seed variance is 2–5pp per arm, 3–6× smaller than the between-arm gap.

Gap: narrow + hard probes on the new 12 models not run

The original presentation's numbers (narrow 25→45, off-task 44→51) come from post-training probes on specific regex tasks, not training-time reward. The 12 new models' weights are in S3 (HF-format exports at exp7-copy_n{N}_seed{S}/exports/exp7-copy_n{N}_seed{S}_<RUN_ID>/global_step_15/policy/) but no probe has been run yet. To directly compare against the original's narrow/hard numbers, someone needs to launch an L4 probe cluster and run the exp6/exp7 narrow + hard probes on those 12 models. That's a ~30-min task on 1 probe cluster.

Status of the original result

Directionally replicated: dose-response is seed-robust in training reward. The original's on-task probe result (monotone 25→45%) and off-task probe result (44→43→51, non-monotone) need fresh probe data on the new 12 models to verify. The interpretation "copy-padding causes dose-dependent capability regression" remains the working hypothesis with 4-seed backing on the in-harness side but unverified on the off-target side.
